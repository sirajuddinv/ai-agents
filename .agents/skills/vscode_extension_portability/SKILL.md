# VS Code Extension Link Portability Rules

This skill provides the protocol for identifying and refactoring ephemeral or machine-specific file paths
generated by VS Code extensions (e.g., Postman, Cursor) within `settings.json` profiles.

***

## 1. Scope & Logic

Extension-generated paths often contain temporary OS cache strings (like `/var/folders/`) or absolute system
prefixes (like `/Users/dk/`). These prevent settings portability and can lead to broken references after
reboots or environment migrations.

### 1.1 Detection Patterns

The agent MUST scan for the following "red flag" patterns in `settings.json`:

- **Temporary Cache**: Paths containing `/var/folders/` or `/tmp/`.
- **Absolute Home Prefixes**: Full hardcoded paths starting with `/Users/` or `C:\Users\`.
- **Extension Ephemeral Storage**: Any path pointing to a hidden system temporary directory that would not
  persist across sessions.

### 1.2 Resolution Standards

- **Home Directory**: Replace user-specific absolute prefixes with the tilde (`~`) character.
- **Extension Permanent Storage**: Map temporary cache locations to the extension's stable internal directory
  (usually `~/.antigravity/extensions/<extension-id>/`).

***

## 2. Environment & Dependencies

Before executing a refactor, the agent MUST verify the environment:

- **Python 3.x**: Required for running the automation script.
- **`settings.json` Backup**: Always create a `.bak` copy before performing automated replacements.

***

## 3. Workflow (The "Refactor Protocol")

1. **Locate `settings.json`**: Identify the current profile's configuration file.
2. **Scan for Non-Portable Links**: Use `grep` or the `refactor_links.py` script to find suspicious paths.
3. **Verify Permanent Alternatives**: If a path points to `/var/folders/`, locate the equivalent file in the
   extension's permanent directory (e.g., `~/.antigravity/extensions/postman.postman-for-vscode-*/`).
4. **Execute Replacement**: Apply the tilde expansion and permanent mapping logic.
5. **Verify Syntax**: Ensure the JSON remains valid and the links are logically correct.

***

## 4. Automation Tooling

The skill includes a dedicated Python script for precise, regex-based refactoring.

### 4.1 Usage: `refactor_links.py`

```bash
python3 .agent/skills/vscode_extension_portability/refactor_links.py --file /path/to/settings.json
```

- `--file`: Path to the target `settings.json`.
- `--dry-run`: (Optional) Log changes without writing to disk.
