## 2. Setting Analysis & Cloud Credentials

### 2.1 The `configureCloudCredentials` Origin

During the cleanup, the setting `configureCloudCredentials` was identified. This setting is exclusively used by the
**Snyk** extension (specifically its vulnerability scanning features) to authenticate against their cloud
infrastructure. Because Snyk was no longer in active use, this credential mapping was purged.

### 2.2 Postman Dynamic AI Instructions (`chat.instructionsFilesLocations`)

Multiple paths targeting `/var/folders/.../T/postman-*.instructions.md` were detected in the
`chat.instructionsFilesLocations` dictionary.

- **Analysis**: These files are dynamically generated by the Postman extension to provide AI context
  for generating `pm.*` scripts. However, they are stored in the macOS temporary cache (`/var/folders/`),
  meaning they are **deleted by the OS** upon reboot or cache clearing.
- **Redundancy**: Permanent copies of these exact instruction files exist within the extension's
  installation directory:
  `~/.antigravity/extensions/postman.postman-for-vscode-*/agent-instructions-files/vscode/`.
- **Recommendation**: The `/var/folders/` entries were **replaced** with permanent paths (using the
  tilde `~` shortcut) pointing directly into the extension's binary installation directory. This
  guarantees that the configuration is **portable** across different machines and ensures AI Chat
  participants always have access to Postman's latest scripting rulesets.

### 2.3 Deep Dive: `editor.suggestSelection`

The `editor.suggestSelection` setting dictates how VS Code pre-selects items in the autocomplete dropdown. Note: It
differs significantly from `vsintellicode.modify.editor.suggestSelection`, which is merely a tracking variable.

- **`first`**: Always selects the top item in the list, regardless of your past choices.
- **`recentlyUsed`**: Memorizes which specific suggestions you previously picked and prioritizes them the next time you
    type the same prefix (Building contextual muscle memory).
- **`recentlyUsedByPrefix`**: Similar to `recentlyUsed`, but strictly considers the exact typed prefix string.

For optimal productivity across files, `recentlyUsed` combined with `editor.suggest.shareSuggestSelections: true` is
the most powerful setup.

## 5. Detailed Explanations

A maximum-detail review of foundational Git, Suggest, and Terminal settings.

### 5.1 `editor.suggest.shareSuggestSelections`

- **Context & Relevance**: When you pick an item from the autocomplete dropdown, VS Code remembers it. Setting this to
    `true` shares that memory across *all* files. It is a massive productivity booster for contextual muscle memory.
- **Alternatives**: Set to `false` if you work in a monolithic repo where different folders use frameworks with rapidly
    overlapping function names (requiring per-file memory isolation).

### 5.2 `git.alwaysSignOff`

- **Context & Relevance**: Automatically appends a `Signed-off-by: Name <email>` trailer to commits made via the Source
    Control panel. Crucial for Open Source contribution compliance (Developer Certificate of Origin / DCO).
- **Alternatives**: Set to `false` (Default) if working purely on private commercial codebases where the trailer creates
    unnecessary git history noise.

### 5.3 `git.autofetch`

- **Context & Relevance**: Silently reaches out to origin servers (like GitHub) every 3 minutes to download new branch
    data. Essential for seeing `↓ 3 ↑ 1` sync indicators in the bottom bar so you know when you're behind.
- **Alternatives**: Set to `false` on metered connections, laptops with low battery, or massive corporate repos where
    background I/O is a bottleneck.

### 5.4 Autofetch vs Pruning

It was clarified that `git.autofetch` does **not** prune deleted remote branches automatically. To enable automatic
pruning during background fetches, configure Git at the system level:

```bash
git config --global fetch.prune true
```

Settings from the Git Graph extension only apply when fetching manually through the extension's UI.

### 5.5 `timeline.pageSize`

- **Context & Relevance**: Controls pagination in the Timeline view, which tracks file history and commits. Setting it
    to `0` mathematically disables pagination entirely, giving an infinite, continuous scroll.
- **Alternatives**: The default value is strictly `50`. Enforce `50` or `100` instead of `0` if you work on ancient
    monolithic repositories where loading the entire endless history visually lags the VS Code UI pane.

### 5.6 `terminal.integrated.unicodeVersion`

- **Context & Relevance**: The modern default is `"11"` (historically `"6"`). Explicitly enforcing `"11"` guarantees
    complex modern emojis, JetBrains font ligatures, and powerline arrows align precisely with proper text spacing.
- **Alternatives**: Revert to `"6"` explicitly only if connecting to extremely legacy remote Linux environments that
    violently truncate characters when encountering modern UTF-8 mathematical width models.

### 5.7 `editor.suggest.localityBonus`

- **Context & Relevance**: Strictly powers VS Code's local, built-in IntelliSense engine (irrelevant to AI engines
    like IntelliCode). Re-ranks variables declared physically closer to your cursor higher than identically-named
    global variables.
- **Alternatives**: Set to `false` in enormous Object-Oriented paradigms (e.g. strict Java or C# architectures) where
    fetching inherited distant global configurations happens exponentially more often than local closure variable hits.

### 5.8 `chat.instructionsFilesLocations`

- **Context & Relevance**: Hardcodes exact mappings to external markdown files (like
  `.github/instructions` or dynamic Postman collections) acting as absolute pre-prompt context.
  It forces AI Chat participants to strictly ingest these explicit rulesets prior to outputting
  responses.
- **Alternatives**: Rely purely on default workspace `.copilot-instructions.md` drops without
  hardcoding absolute ephemeral `/var/folders/` paths that eventually invalidate across reboots.

### 5.9 `debug.allowBreakpointsEverywhere`

- **Context & Relevance**: By bypassing the Language Server's syntax restrictions, this allows
  placing debug execution breakpoints on literally any line. It is a massive lifesaver for
  polyglot scripts (e.g., HTML housing embedded PHP/JS) where native parsers incorrectly assume
  a line is non-executable text.
- **Alternatives**: Leave `false` (default) if you work exclusively in strictly compiled languages
  (Go/C++) so the IDE intelligently blocks you from attempting to break on whitespace or comments.

### 5.10 `debug.inlineValues`

- **Context & Relevance**: Set to `"on"`, debug sessions dynamically evaluate and print the live
  string/integer variables as floating "ghost text" directly adjacent to the exact line of
  execution. You don't need to manually hunt through the "Variables" sidebar.
- **Alternatives**: `"off"` (Default). `"auto"` only shows inline values for variables explicitly
  interacted with on the current line execution step to heavily reduce screen visual clutter.

### 5.11 `debug.showBreakpointsInOverviewRuler`

- **Context & Relevance**: `true` duplicates your active breakpoint red dots directly into the
  miniaturized far-right overview scrollbar, letting you easily scroll immense files directly
  to execution pause triggers.
- **Alternatives**: `false` strictly confines breakpoint markers to the left-hand gutter, heavily
  clearing up scrollbar visual real estate if you map many lint errors natively.

### 5.12 `diffEditor.hideUnchangedRegions.enabled`

- **Context & Relevance**: `true` condenses code comparisons drastically, collapsing and hiding
  hundreds of raw unchanged lines bridging between specific `+` or `-` additions. Perfect for code
  reviews scrolling through massive file diffs.
- **Alternatives**: `false` forces the diff editor to map the entire file verbatim top-to-bottom
  natively, providing maximum surrounding context around a 1-line change.

### 5.13 `diffEditor.ignoreTrimWhitespace`

- **Context & Relevance**: `false` ensures the diff explicitly flags formatting mutations (like
  removing extra spacing at the end of a line) as a legitimate code chunk difference.
- **Alternatives**: `true` silently ignores purely spacing-based differences natively. Highly
  recommended if a teammate forces an automated formatter blindly disrupting thousands of lines
  of whitespace while reviewing the actual logic.

### 5.14 `diffEditor.renderSideBySide`

- **Context & Relevance**: `false` stacks your deleted lines natively directly on top of the
  newly added lines sequentially within a singular "Inline" view model instead of rendering two
  separate split columns.
- **Alternatives**: `true` creates the classic twin-pane structural diff editor, superior on
  ultra-wide monitors where vertical space is less critical natively.

### 5.15 `diffEditor.wordWrap`

- **Context & Relevance**: `"off"` disables wrapping natively inside comparisons. You must
  horizontally scroll to view line endings, preserving structural indentation matching across rows.
- **Alternatives**: `"on"` wraps overflowing code purely visually avoiding horizontal scrolling
  heavily, natively preferred on smaller laptop monitors dynamically.

### 5.16 `editor.accessibilitySupport`

- **Context & Relevance**: `"off"` statically disables heavy internal UI hooks natively optimized
  exclusively for external accessibility screen-reading software.
- **Alternatives**: `"on"` natively enables optimizations mapping explicit visual geometries
  specifically targeting OS-level voice narrators. `"auto"` pings system listeners exclusively.

### 5.17 `editor.formatOnPaste`

- **Context & Relevance**: `false` drops exactly what text you copied unmodified.
- **Alternatives**: `true` automatically detects the language and blindly snaps pasted snippets
  aggressively into compliant formatting configurations, which frequently destroys specific
  hardcoded regex strings instantly.

### 5.18 `editor.formatOnSave`

- **Context & Relevance**: `false` relies exclusively on you explicitly triggering manual format
  commands (`Shift+Option+F`).
- **Alternatives**: `true` automatically mutates formatting structure globally saving the document
  natively. Extreme risk for legacy repositories where linting rules differ wildly.

### 5.19 `editor.formatOnType`

- **Context & Relevance**: `false` stops the editor from radically aggressively shifting trailing
  code immediately after hitting structural characters perfectly cleanly like `;` or `}`.
- **Alternatives**: `true` essentially enforces automatic constant refactoring instantaneously
  on keystrokes, enforcing exact rigid compliance natively dynamically.

### 5.20 `editor.guides.bracketPairs`

- **Context & Relevance**: `false` disables the colored vertical connector lines natively tracing
  between opening and closing `{}` bounds globally.
- **Alternatives**: `true` natively renders bright visual outlines explicitly bounding complex
  nested function architectures.

### 5.21 `editor.guides.indentation`

- **Context & Relevance**: `false` hides the gray dotted trace lines natively connecting massive
  structural indent margins globally natively.
- **Alternatives**: `true` restores the grid system mapping spacing bounds reliably across massive
  JSON objects precisely.

### 5.22 `editor.inlineSuggest.enabled`

- **Context & Relevance**: `true` globally enables the API handling the grey ghost-text predictive
  suggestions universally utilized proactively across AI engines proactively.
- **Alternatives**: `false` universally blinds all LLM inline autocomplete algorithms natively
  preventing popup distractions heavily.

### 5.23 `editor.minimap.enabled`

- **Context & Relevance**: `false` explicitly hides the panoramic minimap overview scrollbar on
  the far right natively reclaiming horizontal screen width natively.
- **Alternatives**: `true` restores the minimap globally, allowing clicking structurally on dense
  code topologies efficiently dynamically mapping visually.

### 5.24 `accessibility.signals.terminalBell`

- **Context & Relevance**: Targets the auditory notification mechanisms within the integrated VS Code terminal.
    Terminal emulators historically use the "bell" character (ASCII `\a` or `07`) to audibly alert the user of
    specific events—such as an interactive script finishing, a compilation hard-failing, or hitting backspace at
    the start of a prompt. Setting this to `"on"` forces VS Code to actively play an audio hardware cue whenever
    the terminal triggers a bell.
- **Developer Impact**: Highly relevant to developers who multitask. If you have a long-running script (like an
    `npm install`, heavy dataset processing, or a `docker-compose boot`) running in a background terminal panel,
    this auditory ping tells you instantly when the terminal demands attention, allowing you to keep your eyes
    on the code while the automation works.
- **Alternatives**: `"off"` silences all terminal pings entirely. `"auto"` (default) only plays sounds when a
    screen reader is definitively detected natively.

### 5.25 `chat.editing.alwaysSaveWithGeneratedChanges`

- **Context & Relevance**: Tied to built-in IDE AI Chat windows (like Copilot Chat or Supermaven).
  Setting explicitly to `true` instantly commands the editor to execute a physical disk Save event
  the microsecond the AI finishes injecting structural diffs. Bypasses manual `Cmd+S` keystrokes,
  instantly triggering hot-reloading web servers or background TDD pipelines seamlessly natively.
- **Alternatives**: `false` (default) leaves the code buffer unsaved, allowing a human "cooling off"
  period to visually diff the hallucinations and perform a manual Undo before committing changes to
  disk securely.

### 5.26 `editor.semanticTokenColorCustomizations`

- **Context & Relevance**: Replaces fragile legacy TextMate scope overrides (like `textMateRules`)
  with intelligent, Language Server-backed token coloring. By explicitly enabling semantic styling,
  you guarantee that symbols like `class` or `variable` maintain consistent, high-contrast visual
  identity across entirely different programming languages natively securely.
- **Alternatives**: Revert to unstructured `editor.tokenColorCustomizations` (TextMate Regex
  parsing) if dealing with obscure legacy languages lacking dedicated AST Language Server support.

### 5.27 `workbench.colorCustomizations`

- **Context & Relevance**: Injected explicitly to protect reference highlight visibility after migrating away from
    `ref.matchtext` TextMate scopes. By forcibly overriding `editor.findMatchHighlightBackground` and
    `peekViewResult.matchHighlightBackground` with custom transparency (`#ffff0055`), it ensures search
    highlight boxes never opaque underlying matched text, guaranteeing strict accessibility regardless of
    default theme defects natively.
- **Alternatives**: Delete these overrides exclusively if switching to rigorously accessible, officially
    maintained high-contrast themes natively lacking overlapping contrast defects.

## 6. Postman Portability Refactor

The Postman extension for VS Code dynamically links to instruction files that guide the AI in generating scripts.
This section documents the transition from fragile temporary paths to robust, portable extension-internal paths.

### 6.1 Temporary Path Fragility

Previously, `settings.json` contained hardcoded mappings to files in `/var/folders/`. These paths were:

1. **User-Specific**: They contained the local system username, breaking configuration portability.
2. **Ephemeral**: Being in the OS temporary directory, the files were periodically purged, causing the AI tool to
   lose its specific Postman context unexpectedly.

### 6.2 Permanent Extension Paths

Investigation revealed that the Postman extension maintains official, permanent copies of these Markdown instructions
within its own installation directory. By pointing directly to these files, we guarantee context persistence:

- **Directory**: `~/.antigravity/extensions/postman.postman-for-vscode-*/agent-instructions-files/vscode/`

### 6.3 Resolution: Tilde Expansion

To ensure the profile remains portable across different dev machines, all hardcoded user-specific prefixes were
stripped and replaced with the tilde (`~`) character. VS Code and the Postman agent correctly expand this to the
current user's home directory.

| Setting File                 | Resolution                                                              |
| :--------------------------- | :---------------------------------------------------------------------- |
| `settings.json`              | Replaced all `/var/folders/` paths with `~/.antigravity/extensions/...` |
| `vscode-settings-cleanup.md` | Documented as the new portability standard.                             |
